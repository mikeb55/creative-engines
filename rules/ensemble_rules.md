# Ensemble Rules

**Version 1.0**  
**Universal — applies to all styles, ensembles, projects**

---

## Rule 4 — Ensemble Awareness (Mandatory)

**Parts must be idiomatic for the ensemble being written for.**

---

## Instrument-Specific Guidelines

| Instrument Family | Idiomatic Requirements |
|-------------------|------------------------|
| **Strings** | Bowable phrasing; sustain considerations; string crossings; playable double-stops |
| **Guitar** | Dyads, triads, chord fragments; not only monophonic lines; playable voicings |
| **Winds** | Breathing space; phrase lengths; register; tonguing |
| **Piano** | Playable hand spans; idiomatic figuration; pedal use |
| **Brass** | Breathing; range; mute changes; endurance |
| **Voice** | Text setting; breathing; range; vowel placement |
| **Percussion** | Stick changes; setup; playable patterns |
| **Electronic** | Parameter ranges; automation; mix considerations |

---

## Adaptive Behaviour

The engine rules adapt automatically to available instruments:

- **Solo instruments:** Single voice; harmonic implication through arpeggiation, double-stops, or register
- **Chamber ensembles:** Dialogue; role distribution; reduced texture options
- **Big band:** Section writing; tutti vs. soli; shout choruses
- **Orchestra:** Tutti, divisi, solo; balance; blend
- **Hybrid ensembles:** Cross-family dialogue; timbral contrast

---

## Dialogue Rule

- Orchestration must show dialogue between instruments.
- No instrument should function in isolation for extended periods.
- Voices respond to each other.

---

## Failsafe

**Regenerate if:** Orchestration lacks dialogue between instruments.
