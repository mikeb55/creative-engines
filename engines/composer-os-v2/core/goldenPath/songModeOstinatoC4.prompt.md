# Phase C4 — Ostinato / hook rhythm (prompt & iteration contract)

Canonical guidance for tuning `songModeOstinatoC4.ts` and related behaviour. **Simpler beats smarter; weaker beats broken.**

---

## Iteration control (mandatory)

### Iteration budget

- **Target:** 2–4 passes  
- **Hard limit:** 5 passes  

If **>5 passes** would be required:

- **STOP** expanding logic  
- **REDUCE** strength and scope instead  

**No** new systems. **No** new layers. **No** complexity escalation.

---

## Known risk zones

- C4 interacting with **C2 timing identity**  
- C4 fighting **C3 groove feel**  
- **Bias vs replacement** misbalance  
- Bass becoming repetitive or **masking groove**  
- Human perception failure (**loop** vs **musical repetition**)

---

## Failure modes (check on first pass)

You **must** explicitly evaluate:

1. **Too weak** — no clear rhythmic fingerprint in bars 2–5  
2. **Too strong** — feels like loop / sequencer / stamped pattern  
3. **Bass failure** — bass as repetitive or more repetitive than melody; bass masks groove clarity  
4. **Mode failure** — Stable ≈ Balanced ≈ Surprise; no clear separation  

---

## Early validation (first pass only)

Check **only**:

- Bars **2–5**  
- **Melody first**, bass second  

Confirm:

- Recognisable **1-bar fingerprint** (when active)  
- **Not** mechanical  
- **Melody** leads pattern  
- **Bass** is lighter  
- **Mode** differences audible  

---

## Allowed fixes (only these)

If failure occurs, adjust **only**:

- **Ostinato strength caps** (onset ≤ 1/16, duration small, accent moderate)  
- **Activation thresholds** (Stable / Balanced / Surprise)  
- **Bass strength scaling** (reduce or disable per phrase if needed)  

**Do not:**

- Add new helpers  
- Add new passes  
- Add new pattern logic  
- Change architecture  

---

## Convergence rule

C4 must converge by:

- Tuning **strength**  
- Tuning **thresholds**  

**Not** by adding logic.

---

## Final goal

Reach a stable state where:

- Ostinato is **clearly audible** but **not dominant**  
- Repetition feels **musical**, not mechanical  
- **Melody > bass** in identity  
- **Modes** are clearly differentiated  
- System remains **simple** and **stable**  

---

## Non-negotiable

If a choice is required:

→ **Simpler** beats smarter  
→ **Weaker** beats broken
